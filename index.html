Box Model

  • Every element is a rectagle
  • Content is whats inside padding is the space between content and border while border is obvious and margin is the space between the border and othe elements
  • By default width and height rules only apply to content but with box-sizing: border-box; it will include padding and border
  • Inspect element shows the diagram
  • Vertical margins between blocks collapse to the largest value
  • A parent and its first/last children can share margins unless padding/borders intervene
  • Use padding to prevent unexpected colapses
  • Use shorthand like margin: 24px 16px; for 24px vertical and 16px horizontal
  • Create as scale and stick to it using custom properties for consistant margins and padding

Layout Fundamentals

  • Blocks stack flow inlines sit in a line and only respect horizontal and inline block flow inline and allow width/height control
  • Switch the flow behavior with display: value;
  • Block makes inline content a full-width block while inline-block makes a box that flows like text my favorite type of display
  • Use float: direction; to take elements out of normal flow
  • Use overflow: auto; to clear the float
  • Modern layouts should use flexbox or grid more but floats are still in old code
  • Elements render in DOM order unless positioning or z-index is applied
  • New stacking contexts form with properties like position + z-index opacity transform or filter
  • Manage stacking contexts to avoid unexpected overlays

Positioning & Z-index

  • Lets you break out of normal flow when you need overlays tooltips sticky headers or UI chrome
  • Positioning has the modes static does nothing relative stays in flow but offsets absolute is out of flow and positioned relative to nearest ancestor with positioning fixed pinned to viewport and ignore scrolling sticky behaves relative until a scroll threshold where it becomes fixed
  • Combine position with offsets to place the elements
  • Sticky headers are often for UI chrome and navigation
  • Only possitioned elements and flex/grid children respect z-index
  • New stacking contexts form when you use position with z-index things like opacity < 1 transform or filter
  • Z-index is like depth
  • Use relative + absolute positioning for badges or icons
  • Keep fixed elements minimal to avoid clutter
  • Use sticky for table headers or inpage navs that follow scrolling
  • Audit stacking context when overlays misbehave inspect element highlights them

Flexbox Layout

  • Flexbox is good for arranging in one dimension either rows or columns while distributing space and aligning the items
  • Flexbox gives simple alignment adaptive spacing and is component friendly
  • Enable flexbox with display: flex; for rows or inline-flex use flex-direction to swap to columns
  • Flexbox in inherited by child elements
  • Main axis follows flex-direction control distrobution with justifty-content
  • Cross axis is peroendicular and aligns items with align-items or per item with align-self
  • Use gap for consistent spacing without extra margins
  • flex: grow shrink and basis; define how items consume space
  • flex-basis sets ideal size before distrobution of free space
  • Combine flex shortcuts like flex: 1; for equal widths or flex: 0 0 300px; for a fixed width
  • The 3 numbers are for grow shrink and base size
  • flex-wrap: wrap; breks items into new rows/colums
  • order reorderes items visualy without changing the DOM

Grid Layout

  • The grid is a 2d layout that defines rows columns and regions useful for dashboards landing pages and complex responsive structure needing minimal markup
  • Enable the grid with display: grid; or display: inline-grid;
  • Use grid-template-columns and grid-template-rows to declare tracks
  • gap or column-gap/row-gap handle track spacing
  • For fixed tracks use something like grid-template-columns: 300px 1fr 1fr;
  • Use fr for one of the numbers to distribute remaining space
  • To make responsive tracks use repeat(auto-fit, minmax(220px, 1fr)) minmax sets the min and max size
  • Auto-placement fills tracks in source order automatically
  • Explicit placement grid-column: 1/7;
  • Use named areas grid-template-areas and grid-area on items
  • Use justify-items and align-items to align content inside tracks
  • Use justify-content and align-content to distribute overall grid within the container
  • place-items and place-content combine bothe axes
  • Use auto-fit/auto-fill with minmax() for fluid cards
  • Combine media queries with track adjustemnts for complex breakpoints
  • Switch to stacked layout by changing grid-template-areas on small screens
  • Fit collapses empty tracks while fill keeps them but both adapt to screen size

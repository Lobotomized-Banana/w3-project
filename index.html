Head Metadata

  • charset defines the character set
  • name="viewport" makes layouts responsive
  • link stylesheet attaches CSS
  • script tags load at the end of the body to not block rendering
  • use title for SEO and the tab name
  • use name="desc" for search snippets

Layout containers

  • header for branding/navigation
  • nav for navigation links
  • main is for the pages unique content
  • section article and aside are subsections with their own heading
  • footer is for closing info links and copyright

Organizing CSS

  • link the css file in the head before the page renders and for critical css that MUST load immediately you can inline it

Accessibility hooks

  • add lang on the html tag for screen readers
  • heading for each section otherwise aria-label
  • keep headings in order h1->h2->h3 to describe hierarchy

Text Content

  • use 1 h1 followed by h2 h3 and on to improve SEO and accessibility this file is a perfect example of what not to do
  • p wraps paragraphs of text
  • br inserts a line break use sparingly
  • hr does thematic breaks
  • strong bolds text
  • em makes text italic
  • mark highlights
  • codemonospace code snippet
  • abbr cite and time add meaning for readers and crawlers
  • use line-height margin and letter-spacing to make more readable

Basic Lists

  • use ul for bullets
  1. and ol for numbers
  • each item goes in the li tag
  • use list-style padding and flexbox to style lists with CSS

Description List

  • dl makes a description list
  • dt is a term
  • and dd is for glossaries or metadata

Tables

thead tbody tr th td
is the tables head is the tables body makes a row makes a header cell makes a data cell

Links & nav

  • make links with the a tag and href attribute
  • For external links use target _blank and noopener noreferrer
  • title gives a optional tooltip
  • download downloads the file

Nav menus

  • use lists to group the links
  • use aria-label to give screen readers a description

States and Transitions

  • .active class to represent current page
  • :hover :focus and :focus-visible help keyboard users(people who can't use a mouse or psycos)
  • use transition for smoother color changes

Responsive Menus

  • For mobile convert horizontal menus into vertical stacks
  • use a menu button and JS for more complex patterns

Images and Media

  • use the img tag with src to add the image and alt to provide a description for screen reades and width is used to set the width taken by the image
  • in the CSS use max-width to set the image to be a percentage of the container width useful for keeping pages consistent across screen size
  • use height:auto to keep the aspect ratio the same when scaled
  • use the picture tag with the source tag to use a different picture by view size
  • use media in the source tag to determine what size the larger img should
  • use srcset to set the img according to media's condition
  • use img on the next line normaly for the smaller picture and giving the alt
  • the audio and video tags allow for the playback of video/audio files natively in browser
  • use poster to display a preview before playback
  • set what type of file is being played with type
  • write a message for if the browser doesnt support media playback natively
  • wrap any video embeds (youtube) in a container to keep aspect ratio
  • use absolute positioning on the iframe to fill the container and 56.25% padding-bottom on the container to keep 16:9 aspect
  • use lazy loading to speed up the loading of the page initally with nothing cached use on images later on the page
  • use figure and figcaption to add accessibility

Forms and Validation

  • use the form tag to wrap the form
  • use action to tell where to send the form
  • use method to use a POST or GET request
  • use the atributes name and autocomplete to help the browser
  • label elements need a id on inputs to work and use a for to specify
  • use required atribute to make the input needed to submit
  • use aria-describedby for helper text
  • group related controls with fieldset and legend
  • use all of the standard CSS to make the form have styling consistant with your page or not
  • use input:valid/invalid to mark valid and invalid inputs with aria-invalid to improve accessibility
  • use grid or flexbox to align labels and inputs
  • Use HTTPS so data is encrypted and use server side validation and prevent CSRF and XSS attacks when handleing submissions and SQL injection if you use SQL so sanitize your inputs

Reuseable Components

  • Use meaning full class names
  • Scope styles to prevent side effects
  • for buttons use a base with CSS consistant across all buttons and then add variants to the base
  • makeing all of my css into reuseable components was a pain but now it works well
